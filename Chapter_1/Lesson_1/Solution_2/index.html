Thoughts on the solution

This solution uses the jQuery .siblings() function to find the input and the text output fields. You can see the usage of filtering selectors to the siblings as well. Those restrictions work like normal jQuery selectors but always on the context of the element where the function is called.

It is a nice approach but only works when the text output is actually a sibling of the clicked button. The advantage to the first solution is that we could add an additional input with the same markup and it will work without changing the script. But if the text is anywhere else but a sibling of the button it won't work anymore.

  • What have we done to make it work?
  • Attaching the event handler to every button
  • Fetching the value of the input which is a sibling to the button
  • Writing the value to the span.helper-text which is a sibling of the button
