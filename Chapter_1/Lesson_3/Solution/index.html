Thoughts on the solution

This solution retrieves the current text and compares it to the text entered. Only when it has been changed the counter increase and the update will be executed.

This approach uses the comparison by !== which compares typesafe. When 1 == '1' will result in true 1 === '1' would result in false since one is a number and the other one is a string. In that case it wouldn't matter since both variables are strings. But it is always safe to compare typesafe. The same comparison rule applies to the negation (!= versus !==).

The advantages to the previous solution is that the update of the DOM will only be made when an actual change has been made.
